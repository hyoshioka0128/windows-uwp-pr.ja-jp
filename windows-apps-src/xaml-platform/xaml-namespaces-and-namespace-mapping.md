<span data-ttu-id="4eb62-227">XAML で使うコードを定義するときに入れ子になった型を使わないでください (クラス内で列挙型を入れ子にするなど)。</span><span class="sxs-lookup"><span data-stu-id="4eb62-227">Don't use nested types (such as nesting an enumeration within a class) when you define your code for use with XAML.</span></span> <span data-ttu-id="4eb62-228">入れ子になった型は評価できません。</span><span class="sxs-lookup"><span data-stu-id="4eb62-228">Nested types can't be evaluated.</span></span> <span data-ttu-id="4eb62-229">ドットが名前空間名の一部なのか、それとも入れ子になった型名の一部なのかを区別する方法は XAML パーサーにはありません。</span><span class="sxs-lookup"><span data-stu-id="4eb62-229">There's no way for the XAML parser to distinguish that a dot is part of the nested type name rather than part of the namespace name.</span></span>

<span data-ttu-id="4eb62-235">主要なアプリのアプリケーション定義やページ定義のカスタム型を参照する場合は、それらの型を使うのにさらに依存アセンブリを構成する必要はありませんが、それらの型を含むコード名前空間をマップする必要はあります。</span><span class="sxs-lookup"><span data-stu-id="4eb62-235">If you are referencing custom types from the primary app's application definition or page definitions, those types are available without further dependent assembly configuration, but you still must map the code namespace that contains those types.</span></span> <span data-ttu-id="4eb62-236">通常は、プレフィックス "local" を特定の XAML ページの既定のコード名前空間にマップします。</span><span class="sxs-lookup"><span data-stu-id="4eb62-236">A common convention is to map the prefix "local" for the default code namespace of any given XAML page.</span></span> <span data-ttu-id="4eb62-237">この方法は、XAML プロジェクトのプロジェクト開始テンプレートでよく使われています。</span><span class="sxs-lookup"><span data-stu-id="4eb62-237">This convention is often included in starting project templates for XAML projects.</span></span>

## <a name="attached-properties"></a><span data-ttu-id="4eb62-238">添付プロパティ</span><span class="sxs-lookup"><span data-stu-id="4eb62-238">Attached properties</span></span>

<span data-ttu-id="4eb62-239">添付プロパティを参照する場合、添付プロパティ名の所有者型の部分は、既定の XAML 名前空間に存在するか、またはプレフィックスが付けられている必要があります。</span><span class="sxs-lookup"><span data-stu-id="4eb62-239">If you are referencing attached properties, the owner-type portion of the attached property name must either be in the default XAML namespace or be prefixed.</span></span> <span data-ttu-id="4eb62-240">要素とは別に属性にプレフィックスを付けることはまれですが、これは、特にカスタム添付プロパティの場合に、必要となることのあるケースの 1 つです。</span><span class="sxs-lookup"><span data-stu-id="4eb62-240">It's rare to prefix attributes separately from their elements but this is one case where it's sometimes required, particularly for a custom attached property.</span></span> <span data-ttu-id="4eb62-241">詳しくは、「[カスタム添付プロパティ](custom-attached-properties.md)」をご覧ください。</span><span class="sxs-lookup"><span data-stu-id="4eb62-241">For more info, see [Custom attached properties](custom-attached-properties.md).</span></span>
